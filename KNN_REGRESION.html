Variando el hiperparmetro $k$: bsqueda por grilla

En un modelo kNN el nmero de $k$ de vecinos determina la flexibilidad e interpretabilidad del mismo, y con ello, la potencia predictiva sobre nuevos conjuntos de datos. Cada $k$ determina un modelo particular, entonces la tarea se traduce en entrenar distintos kNN para distintos valores de $k$, mientras se evalan sus correspondientes desempeos predictivos sobre el conjunto de prueba $P$. A esta estrategia se le denomina bsqueda por grilla del hiperparmetro de entrenamiento ptimo $k$ ($k$ es un hiperparmetro que desea encontrarse, pero que no es estimado por el propio algoritmo):

In[117]:
# Establecer una semilla para la reproducibilidad
np.random.seed(483)

# Definir los valores de k (1, 2, 3,..., 100)
k_values = range(1, 101)

# Vector vaco para almacenar los RMSE de entrenamiento
rmse_e = []

# Bucle para calcular RMSE para cada valor de k
for k in k_values:
    # Crear y ajustar el modelo kNN para el valor de k actual
    knn = KNeighborsRegressor(n_neighbors=k)
    knn.fit(train_set.drop(columns=['fcma']), train_set['fcma'])

    # Realizar las predicciones en el conjunto de entrenamiento
    pred_e = knn.predict(train_set.drop(columns=['fcma']))

    # Calcular el RMSE para el conjunto de entrenamiento
    rmse_k = np.sqrt(mean_squared_error(train_set['fcma'], pred_e))

    # Almacenar el RMSE para el valor de k
    rmse_e.append(rmse_k)

# Mostrar los primeros 10 valores de RMSE (entrenamiento)
print(rmse_e[:10])

Examples
--------

    The simplest way to use nbconvert is

            > jupyter nbconvert mynotebook.ipynb --to html

            Options include ['asciidoc', 'custom', 'html', 'latex', 'markdown', 'notebook', 'pdf', 'python', 'qtpdf', 'qtpng', 'rst', 'script', 'slides', 'webpdf'].

            > jupyter nbconvert --to latex mynotebook.ipynb

            Both HTML and LaTeX support multiple output templates. LaTeX includes
            'base', 'article' and 'report'.  HTML includes 'basic', 'lab' and
            'classic'. You can specify the flavor of the format used.

            > jupyter nbconvert --to html --template lab mynotebook.ipynb

            You can also pipe the output to stdout, rather than a file

            > jupyter nbconvert mynotebook.ipynb --stdout

            PDF is generated via latex

            > jupyter nbconvert mynotebook.ipynb --to pdf

            You can get (and serve) a Reveal.js-powered slideshow

            > jupyter nbconvert myslides.ipynb --to slides --post serve

            Multiple notebooks can be given at the command line in a couple of
            different ways:

            > jupyter nbconvert notebook*.ipynb
            > jupyter nbconvert notebook1.ipynb notebook2.ipynb

            or you can specify the notebooks list in a config file, containing::

                c.NbConvertApp.notebooks = ["my_notebook.ipynb"]

            > jupyter nbconvert --config mycfg.py

To see all available configurables, use `--help-all`.
